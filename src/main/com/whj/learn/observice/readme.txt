观察者模式：

    概念：又称为发布-订阅模式，它定义了一种一对多的依赖关系，让多个观察者同事监听某一主题，当主题
          状态变化时，通知所有的观察者对象，使他们能自动更新自己。


    优点：观察者和被观察者是抽象耦合的

    角色：
       1、抽象观察者对象：定义了观察者的标准方法
       2、抽象被观察者：定义了被观察者对象的标准方法(一般是注册观察者，移除观察者和通知观察者几个方法)
       3、具体的观察者对象：实现观察者对象，定义了具体的处理方法
       4、具体的被观察者对象：实现了被观察者对象，定义了具体的处理方法，同时定义了一个List用来存放观察者对象

       当被观察者属性有变动时，通知观察者进行相应的操作
