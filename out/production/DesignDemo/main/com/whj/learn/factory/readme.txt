工厂模式：

1、简单工厂模式：又叫静态工厂模式；

    特点就是一个实体工厂根据传入的参数来决定创建哪一种产品

    组成部分：
        1、工厂类：这是核心，根据不同类型，创建产品
        2、抽象产品类：是具体产品继承的父类或实现的接口
        3、具体的产品类：需要创建的对象

2、工厂方法模式：一个抽象工厂类可以派生出多个具体工厂类，每个具体工厂只能创建一个具体的产品实例

    特点就是一个抽象工厂派生多个实例工厂，每个实例工厂只能创建一个具体实例，每次需要增加新的产品类型，
       都需要新创建一个实例工厂

    组成部分：
        1、抽象工厂角色：这是核心，是具体工厂必须实现或继承的父类
        2、具体工厂角色：用来创建对应的具体实例
        3、抽象产品角色：是具体产品继承或实现的父类
        4、具体的产品角色：需要创建的目标对象

3、抽象工厂模式：

        抽象工厂模式是工厂模式的升级，工厂模式中抽象工厂中只有一个产品登记，而抽象工厂模式则包含多个产品等级
