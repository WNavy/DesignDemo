单例模式：保证只有一个实例存在。其特点如下：
   1、私有构造方法
   2、私有的静态实例
   3、静态的实例方法

单例模式的的几种形式：
    ①饿汉模式：先实例化对象，在类创建的同时就创建了好了一个对象供系统使用，以后不再改变，
              天生是线程安全的。
    ②懒汉模式：等需要的时候再创建实例对象

对于懒汉模式：是非线程安全的，见test2,如何实现懒汉模式线程安全：
    第一：使用关键字synchronized

单例模式的优点：
1、确保所有的对象都访问同一个实例
2、由于系统内存中只存在一个对象，因此可以节约系统资源
   ，当需要频繁创建和销毁对象时，单例模式可以提高系统的性能。


适用场景：
1、需要频繁创建和销毁的对象；
2、日志文件，应用配置，线程池

一般建议使用饿汉模式，或者使用静态内部类方式的单例模式
